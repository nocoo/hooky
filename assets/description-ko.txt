Hooky — 원클릭 Webhook 트리거, 페이지 컨텍스트 자동 전달, 스마트 규칙 지원.

Webhook 템플릿을 만들고, 현재 페이지 데이터를 자동으로 채워서 도구 모음, 우클릭 메뉴 또는 빠른 전송 규칙으로 원클릭 전송하세요.

🪝 WEBHOOK 템플릿
이름이 지정된 여러 Webhook 템플릿을 만들 수 있습니다. 각 템플릿에 URL, HTTP 메서드(GET/POST/PUT/PATCH/DELETE), 키-값 매개변수를 개별 설정 가능.

🔀 템플릿 변수
템플릿 변수를 사용하여 실시간 페이지 정보를 Webhook 매개변수에 삽입:
• {{page.url}} — 현재 페이지 URL
• {{page.title}} — 페이지 제목
• {{page.selection}} — 선택한 텍스트
• {{page.meta.description}} — 메타 설명
• {{page.meta.og:title}} — Open Graph 제목
• {{page.meta.og:description}} — Open Graph 설명
• {{page.meta.og:image}} — Open Graph 이미지 URL

⚡ 빠른 전송 규칙
규칙을 정의하여 현재 페이지에 따라 전송할 Webhook 템플릿을 자동 선택합니다. 도구 모음 아이콘을 클릭하면 규칙이 순서대로 평가되며, 첫 번째로 일치하는 규칙의 Webhook이 즉시 전송됩니다 — 팝업이 열리지 않습니다.

규칙 조건: URL 또는 제목 × 포함, 일치, 시작, 끝, 정규식 매칭. 우선순위 순서 조정, 개별 활성화/비활성화 가능. 어떤 규칙과도 일치하지 않으면 평소처럼 팝업이 열립니다.

배지 피드백: ✓(녹색) 전송 성공, ✗(빨간색) 전송 실패.

📋 컨텍스트 메뉴
아무 페이지에서 우클릭하면 "Hooky" 메뉴에 모든 템플릿이 표시되며, 한 번의 클릭으로 전송할 수 있습니다.

🎨 테마
시스템, 라이트, 다크 테마 중 선택 가능.

🌐 언어
10개 언어 지원: English, 简体中文, 繁體中文, 日本語, 한국어, Français, Deutsch, Español, Português (BR), Русский.

🔒 개인정보
Hooky는 사용자 데이터를 수집, 저장 또는 전송하지 않습니다. 모든 설정은 브라우저 로컬(chrome.storage.local)에 저장됩니다. Webhook 요청은 사용자가 설정한 URL로만 전송됩니다.

활용 사례
• 기사를 Notion, Raindrop 또는 원하는 API에 저장
• 페이지 URL을 Slack, Discord 또는 Telegram 봇에 전송
• 페이지 메타데이터로 CI/CD 파이프라인 트리거
• 브라우징 활동을 개인 Webhook에 기록
• 페이지를 커스텀 백엔드에 북마크

권한
• activeTab — 템플릿 변수용 현재 페이지 URL 및 제목 읽기
• storage — Webhook 템플릿과 규칙을 로컬에 저장
• contextMenus — 우클릭 메뉴에 "Hooky" 추가
• 호스트 권한(모든 URL) — 설정한 모든 URL로 Webhook 요청을 보낼 수 있도록 합니다. 대부분의 Webhook 엔드포인트는 서버 간 통신용으로 설계되어 브라우저의 CORS 사전 요청을 처리하지 않습니다. 이 권한이 없으면 JSON 본문이 포함된 POST 요청이 브라우저에 의해 차단됩니다. Hooky는 명시적으로 설정한 URL로만 요청을 보냅니다
